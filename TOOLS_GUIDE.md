# HTML Tools Repository Guide

This repository is for small, static, browser-based tools written as standalone HTML files.

## Layout

```
htmltools/
├── *.html              # Individual tools at the repository root
├── *.docs.md           # Short documentation for each tool
├── footer.js           # Shared footer injected into tool pages
├── gather_links.py     # Builds tool metadata from HTML/docs/git history
├── build_dates.py      # Builds dates.json for footer update labels
├── build_index.py      # Generates index.html from README.md and tools.json
├── build.sh            # Local build pipeline
├── tests/              # Python tests for build scripts
├── site-config.json    # Site metadata and optional repository links
└── .github/workflows/  # CI and Pages deployment
```

## Conventions

- Name tools as `{slug}.html`.
- Add matching docs as `{slug}.docs.md`.
- Keep tools self-contained unless shared code is clearly worth it.
- Prefer client-side code so everything can be hosted as static files.
- Make pages responsive and usable on mobile.

## Tool docs

Each `*.docs.md` should start with a short paragraph describing the tool. `gather_links.py` uses the first paragraph as the homepage description.

Example:

```markdown
This tool previews HTML in a sandboxed iframe while preserving the source text for editing.
It is useful for quick experiments, demos, and validating snippets generated by AI tools.
```

## Build flow

`./build.sh` runs:

1. `gather_links.py`
2. `build_dates.py`
3. `build_index.py`

The output files are:

- `tools.json`
- `dates.json`
- `index.html`

## Adding a new tool

1. Create `my-tool.html`
2. Create `my-tool.docs.md`
3. Add the tool to `README.md`
4. Run `./build.sh`
